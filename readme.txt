--config/层:存放配置文件以及 API 接口数据，把所有的项目的配置和接口数据均放在这里，
            用 python支持较好的配置文件格式 ini进行配置，实现配置和数据与代码分离。
            当然也可以使用 excel 或者数据库存放接口数据）；
--log/层：  存放生成的日志文件，包括运行日志 Runtime.log和错误日志 Error.log等，
            logger名称为每个测试类的类名。(我是把所有的日志放在了一个文件中，
            当然根据具体需要，也可以每个测试类生成一个日志文件)；
--venv/层： 运行所依赖的库；
--data/层：存放测试过程中写入的文件，如：测试过程中涉及到的一些数据的存取文件；
--report/层：存放程序运行生成的 html格式报告；
--src/源码层: 存放所有程序代码。其中还需要更进一步的分层：
    --common/：公用类，提供一些公共的方法；
        --drivers：存放所需的驱动，如Chromedriver、IEDriverServer等；
    --testcase/：测试用例层； 所有的测试用例类，所有的用例都需要以：’test_*‘开头或‘*_test’结尾；
    --page/： 针对测试页面的元素操作，为测试用例类提供服务，如提供测试数据，也可以在该层增加一些其他公用方法；
--HTMLTestRunner.py : 生成测试报告的依赖；
--run.py：启动执行文件；
